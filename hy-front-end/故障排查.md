# 页面不显示 - 故障排查指南

## ❌ 问题：Chrome中页面不显示

### 原因 1：开发服务器未启动 ⭐ 最常见

**解决方案**：

1. **方法一**：双击运行 `启动开发服务器.bat`

2. **方法二**：在终端/PowerShell中运行：
   ```bash
   cd D:\GDMCP
   npm run dev
   ```

3. 等待看到类似以下输出：
   ```
   VITE v5.0.8  ready in 500 ms

   ➜  Local:   http://localhost:5173/
   ➜  Network: http://192.168.x.x:5173/
   ```

4. 在Chrome中打开：`http://localhost:5173/`

---

### 原因 2：端口被占用

**检查端口**：
```bash
netstat -ano | findstr :5173
```

**解决方案**：
- 如果端口被占用，修改 `vite.config.js` 中的端口号
- 或者杀掉占用端口的进程

---

### 原因 3：依赖未安装

**检查**：查看是否有 `node_modules` 文件夹

**解决方案**：
```bash
npm install
```

---

### 原因 4：浏览器缓存问题

**解决方案**：
1. 按 `Ctrl + Shift + Delete` 清除缓存
2. 或者按 `Ctrl + F5` 强制刷新
3. 或者使用无痕模式打开

---

### 原因 5：JavaScript错误

**检查方法**：
1. 按 `F12` 打开开发者工具
2. 查看 Console 标签页是否有红色错误
3. 查看 Network 标签页，检查资源加载情况

**常见错误**：
- ❌ `Failed to fetch` - 开发服务器未启动
- ❌ `Module not found` - 依赖问题，运行 `npm install`
- ❌ `Unexpected token` - 代码语法错误

---

## ✅ 正常启动的标志

启动成功后，您应该看到：

1. **终端输出**：
   ```
   ➜  Local:   http://localhost:5173/
   ```

2. **Chrome显示**：
   - 登录页面（首次访问）
   - 或主页（已登录）

3. **控制台无错误**：
   - 按 F12 打开控制台
   - Console 标签页无红色错误

---

## 🔧 快速诊断命令

### 1. 检查 Node 和 npm 版本
```bash
node -v
npm -v
```
应该显示版本号（Node >= 16）

### 2. 检查开发服务器进程
```bash
Get-Process | Where-Object {$_.ProcessName -like "*node*"}
```

### 3. 检查端口占用
```bash
netstat -ano | findstr :5173
```

### 4. 重新安装依赖
```bash
rm -rf node_modules
npm install
```

---

## 📞 仍然无法解决？

请提供以下信息：

1. **终端输出**：运行 `npm run dev` 后的完整输出
2. **浏览器控制台**：F12 -> Console 标签页的错误信息
3. **Network 状态**：F12 -> Network 标签页，查看是否有失败的请求
4. **Node 版本**：`node -v` 的输出

---

## 💡 提示

- 开发服务器必须保持运行状态
- 不要关闭终端窗口
- 每次修改代码后，页面会自动刷新（热更新）
- 如果看到空白页，先检查控制台错误
