# 推送服务配置指南

## 问题描述
当前推送服务 [https://push.spug.cc/send/E3w5LmlZzZrxYea4](https://push.spug.cc/send/E3w5LmlZzZrxYea4) 返回：
```json
{"code": 429, "msg": "IP限流，请稍后再试"}
```

## 解决方案

### 1. 访问推送服务管理页面
访问：https://push.spug.cc/

### 2. 登录或注册账号
- 如果没有账号，请先注册
- 如果已有账号，请登录

### 3. 找到推送通道
- 查找推送通道ID：`E3w5LmlZzZrxYea4`
- 进入该通道的配置页面

### 4. 检查模板配置
- 确认模板编码正确
- 检查模板参数格式
- 验证模板是否已启用

### 4. 配置推送对象
在推送通道配置中添加以下手机号：
- `13800138001`
- `18830364127`
- 或其他需要接收验证码的手机号

### 5. 配置推送方式
- 选择短信推送
- 设置推送模板（如果需要）
- 保存配置

### 6. 测试配置
配置完成后，重新测试验证码发送功能

## 配置完成后的效果
- 推送服务将返回 `{"code": 200, "msg": "发送成功"}`
- 手机将收到包含验证码的短信
- 验证码将不再返回给客户端，只发送到手机

## 备用方案
如果无法配置推送服务，可以：
1. 使用其他短信服务商（阿里云、腾讯云等）
2. 修改配置文件中的 `verification.api.url`
3. 在 `SmsService.java` 中实现具体的短信发送逻辑

## 注意事项
- 确保手机号格式正确（11位数字，以1开头）
- 推送服务可能有发送频率限制
- 验证码有效期为5分钟
